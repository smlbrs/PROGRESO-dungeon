# Carpeta include

Esta carpeta contiene las interfaces y definiciones de clases del proyecto.


# ¿Para qué se implementó?

Se implementó para declarar estructuras, atributos y funciones utilizadas por las diferentes clases del videojuego.

# ¿Por qué se consideró necesario?

Separar las interfaces de las implementaciones mejora:

organización, claridad del código

# ¿Cómo se llevó a cabo su implementación?

Cada archivo .h declara:

- atributos
- métodos
- relaciones entre clases

Ejemplos:

- juego.h
- jugador.h
- sala.h
- inventario.h

# Innovaciones declaradas

## Sistema modular

Cada componente del juego fue separado en clases independientes.

## Encapsulamiento

Los atributos y métodos se organizaron siguiendo principios básicos de POO.

## Escalabilidad

La estructura permite agregar nuevas salas, enemigos y mecánicas fácilmente.
